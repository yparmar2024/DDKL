DDKL Syntax Guide

1. Layout

Define the chart type and its orientation.

<graph>: <orientation>

Valid graphs: Flowchart

Valid orientations: top_bottom, bottom_top, left_right, right_left

2. Subgraphs

Declare subgraphs and group nodes.

Subgraph <id> [<name>] ... nodes ... end

Valid ids: Use snake_case to append words, i.e., "yes_workflow_step_2"

3. Nodes

Declare nodes with a unique ID, a shape type, and the text to display.

Node <id> <type>: <description>

Valid ids: Use snake_case to append words, i.e., "yes_workflow_step_2"

Valid types:

  • step: Creates a rectangular border, used for standard process steps
  • decision: Creates a rhombus border, used for branching points
  • start: Creates a capsule border, used for starting processes
  • end: Creates a capsule border, used for ending processes
  • document: Creates a parallelogram border, used for required documents
  • db: Creates a cylinder border, used for databases

4. Arrows

Declare arrows to connect nodes by their IDs, optionally include a label in quotes.

Arrow <type> [<label>]: <start_id>, <end_id>

Valid types:

  • solid: Creates a solid arrow, used for progression
  • dotted: Created a dotted arrow, used for sending information

Valid ids: Use snake_case to append words, i.e., "yes_workflow_step_2"

5. Comments

Declare comments starting with # to use for notes.
